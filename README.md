# SDLC (Software Development Life Cycle): SDLC is the step-by-step process to build software.

1.   Requirement Gathering  
   – What do we need to make?

2.   Design  
   – How will the system look and work?

3.   Development  
   – Writing the actual code.

4.   Testing  
   – Checking for bugs and errors.

5.   Deployment  
   – Releasing the software to users.

6.   Maintenance  
   – Fixing issues and adding updates after release.

In simple:  
SDLC = Plan → Build → Test → Release → Maintain.

-------------------------------------------------------------------------------------------------------------------------------------------------

# Agility (Agile Method): Agile is a modern, fast, flexible way of building software.

-Work is done in small parts (sprints).
-Teams deliver small updates frequently.
-Easy to make changes anytime.
-Continuous communication with the client.

In simple:  
Agile means doing work in small steps, improving constantly, and adapting quickly.

# Required Installations (Developer Essentials): simple reason for why each tool is needed.



# Node.js – Install via NVM  

* Node.js   → Run JavaScript outside the browser (backend apps, tools).
* NVM (Node Version Manager)   → Allows installing multiple versions of Node.

(Used for JavaScript development and DevOps tools)

-----------------------------------------------------------------------------------------------------------------------------------------------------

# Java – Install JVM / JDK  

*   JVM   → Runs Java programs.
*   JDK   → Tools needed to compile Java code.

Used for enterprise apps, Android tools, Jenkins, and backend services.*

-----------------------------------------------------------------------------------------------------------------------------------------------------

# Python – Install using pyenv  

*   Python   → Scripting, automation, ML/AI, DevOps.
*   pyenv   → Manage multiple Python versions.

(Good for automation and cloud scripts.)

-------------------------------------------------------------------------------------------------------------------------------------------------

# Terraform – TFSwitch  

*   Terraform   → Infrastructure as code (cloud resources).
*   TFSwitch   → Quickly switch Terraform versions.

(Used in DevOps to manage cloud systems).

------------------------------------------------------------------------------------------------------------------------------------------------

# Ansible  

* Automation tool for configuring servers.
* No agent needed; works through SSH.

(Used for auto-config, deployments, patching).

--------------------------------------------------------------------------------------------------------------------------------------------------

# AWS CLI  

* Command-line tool to control AWS.
* Used to create S3 buckets, EC2, IAM, Lambda, etc.

(Essential for cloud engineering).

--------------------------------------------------------------------------------------------------------------------------------------------------

# Windows → Use WSL (Windows Subsystem for Linux)  

* Gives you a   Linux environment inside Windows.
* Lets you use Ubuntu, bash, Linux commands, DevOps tools.

(Very useful for Node, Python, Terraform, Ansible, AWS CLI.)

---------------------------------------------------------------------------------------------------------------------------------------------------

# VS Code – Vim/Nano  

*   VS Code   → Modern code editor.
*   Vim/Nano   → Terminal-based editors (useful for servers).

(VS Code for comfort, Vim/Nano for fast editing on servers).

----------------------------------------------------------------------------------------------------------------------------------------------------

# Git / GitHub  

*   Git   → Version control (track changes in code).
*   GitHub   → Cloud platform for sharing repositories.

(Used for collaboration and DevOps pipelines.)

---------------------------------------------------------------------------------------------------------------------------------------------------
